Endpoints¶

Login¶

/api/rest/v001/login/ - Login endpoint¶

POST /api/rest/v001/login/¶

Create an authenticated session for user. This session is required for using private API. If response status code is 200, user is logged in successfully.

Request JSON Object:
 
  • email (string) – User email
  • password (string) – User password
 
POST /api/rest/v001/login/ HTTP/1.1
Content-Type: text/javascript

{'password': 'SeCrEt', 'username': 'staff_user@gettipsi.com'}

HTTP/1.1 200 OK
Content-Type: text/javascript

{}

Full text search¶

/api/rest/v001/fts/ - Search products¶

GET /api/rest/v001/fts/¶

To filter products by product_type just add that type (wine or drink) to fts_fields. (For instance to get only wines you should add wine to fts_fields and do not add drink to fts_fields and do not add drink_fields query param)

Query Parameters:
 
  • fts_fields – (str list) fields from FTSSerializer
  • order_by –

    (str list) fields to order by. (e.g. ‘-vintage’). Sign ‘-‘ revers sort direction. Default ordering id (-rank,id in case query param is passed to request). Available fields:

    • vintage - wine vintage (doesn’t make sence for drinks)
  • query – (str) Full text search.
  • wine_id – (int list) Filter by wine_id (can be used with drink_id, in such case endpoint returns wines and drink together. See example 3)
  • drink_id – (int list) Filter by drink_id (can be used with wine_id, in such case endpoint returns wines and drink together. See example 3)
  • tag – (int list) Filter drinks by drink_tag id
  • producer_id – (int list) Filter drinks by producer
  • drink_type –

    (str list) Filter drinks by drink_type. Available drink_type:

    • beer
    • cocktail
    • spirits
    • other
  • varietal_id – (int list) Filter wines by varietal id
  • sub_region_id – (int list) Filter wines by subregion id
  • winery_id – (int list) Filter wines by winery id
  • vineyard_id – (int list) Filter wines by vineyard id
  • designation_id – (int list) Filter wines by designation id
  • region_id – (int list) Filter wines by region id
  • vintage – (int|NV list) Filter wines by vintage (NV for non-vintage)
  • color – (int list) Filter wines by color
  • wine_type –

    (str list) Filter wines by type. Available types:

    • regular
    • fortified
    • sparkling
    • dessert
    • offdry
  • pro_rating_from – (int) Filter wines by min pro_rating
  • pro_rating_to – (int) Filter wines by max pro_rating
  • country_id – (int list) Filter wines and drinks by country id
  • country_id_exclude – (int list) Filter wines and drinks which are not in country id

Search API requires at least one of next http query params:

  • query
  • wine_id
  • drink_id
  • winery_id

POS Sync¶

/api/rest/v001/store/<store_id>/ext/sync - Sync Inventory¶

PATCH /api/rest/v001/store/(int: store_id)/ext/sync¶

Body must be JSON encoded array list of inventory structs. Each struct has to contain “external_id” param, which will be used to lookup related item in inventory. Unnecessary params can be omitted, in that case they will not be updated. By the way, as “external_id” is used to lookup inventory, inventory Tipsi “id” param is prohibited here as the request will look ambiguously, thus trying to use it will cause error with HTTP respone 400.

NOTE: you need to specify “category” field on item to be either: “wine” or “drink” to create tasks with correct types

Parameters:
  • store_id – (int) Store id
Request JSON Object:
 
  • [] – list of structs with fields from RetailInvExtIdBulkCreateSerializer or RetailInvExtIdBulkUpdateSerializer

/api/rest/v001/store/<store_id>/ext/sync_clear - Sync Inventory With Clearing¶

PATCH /api/rest/v001/store/(int: store_id)/ext/sync_clear¶

Same, as /api/rest/v001/store/<store_id>/ext/sync - Sync Inventory, except will clear inventory not listed in the batch. It’s a safe method as it will just mark in_stock parameter to 0. For real inventory deletion DELETE method should be used upon each inventory item.

Parameters:
  • store_id – (int) Store id
Request JSON Object:
 
  • [] – list of structs with fields from RetailInvExtIdBulkCreateSerializer or RetailInvExtIdBulkUpdateSerializer

Orders API¶

A user must have admin privileges to fetch or modify store orders.

api/rest/v001/group/<group_id>/order_management/order/ - List of orders for retail group¶

GET /api/rest/v001/group/group/(int: group_id)/order_management/order/¶

Return paginated orders list of all the stores in the given retail group. If you want to limit results to the given store, it’s required to pass store_id into query parameters.

Parameters:
  • group_id – (int) Retail group id
Query Parameters:
 
  • query – (str) Filter order by search string. Do full/partial/match with typos for following fields: code, user name, phone
  • order_fields – (str list) List of requested order fields from OrderSerializer
  • store_fields – (str list) List of requested order store fields from StoreSerializerIntegration
  • store_id – (int) Limit results to the given store_id, optional parameter
  • page – (int) Page number starting from “1”, optional parameter
  • page_size – (int) The number of results per a page, optional parameter
  • order_status – (str list) Filter orders by order_status. For statuses see OrderSerializer
  • order_status_exclude – (str list) Exclude orders with given order_status. For statuses see OrderSerializer
  • payment_status – (str list) Filter orders by payment_status. For statuses see OrderSerializer
  • updated_from – (str) Show orders updated from given time. (e.g. 2018-05-05 8:00)
